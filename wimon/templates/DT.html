Directory Traversal 体験サイト

■ 説明（Directory Traversal とは）
Directory Traversal（ディレクトリトラバーサル）とは、 ファイル名を指定する処理に対して ../ を含むパスを送信し、 本来アクセスできないファイルを読み取ってしまう脆弱性です。
入力値の検証不足が原因で発生します。
■ 使い方（攻撃体験方法）
  1. まず「正常なアクセス例」をクリックします
  2. 次に「ディレクトリトラバーサル攻撃」をクリックします
  3. URLバーの file= の値を変更してみてください
※ ../ を増やすと、より上の階層へ移動できます
■ 正常なアクセス例（安全）
/DT/read?file=readme.txt
■ ディレクトリトラバーサル攻撃
/DT/read?file=../secret.txt
■ さらに深い攻撃（別フォルダ）
/DT/read?file=../../address.txt
■ 自分で URL を変えて試してください
  • /DT/read?file=../../../config.ini
  • /DT/read?file=../../../../staff.txt
■ なぜ攻撃が成功するのか
サーバー側で以下のような実装をしている場合、 Directory Traversal 攻撃が成立します。
  • ファイル名をそのままパスに連結している
  • ../ をチェックしていない
  • アクセス可能なディレクトリ制限がない
■ 対策方法
実際のシステムでは、以下の対策が必要です。
  • 許可したファイル名のみ扱う（ホワイトリスト）
  • ../ や / を含む入力を拒否
  • 実ファイルパスを直接指定させない
  • Web公開ディレクトリ外のファイルを読ませない

⚠️ このサイトは学習用です。
実際のシステムに対して行うと違法です。
